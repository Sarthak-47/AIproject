==================================================
SYSTEM ARCHITECTURE: EARNINGS SURPRISE PREDICTOR
==================================================

1. OVERVIEW
-----------
The Earnings Surprise Predictor is a full-stack financial technology application 
that forecasts whether a company will "beat" or "miss" its earnings expectations. 
It combines real-time financial data, machine learning insights, and a 
weighted analytical scoring system to provide actionable confidence scores.

2. SYSTEM COMPONENTS
--------------------

A. FRONTEND (UI LAYER)
   - Framework: React.js + Vite.
   - Styling: Tailwind CSS (Custom "Glassmorphism" Design).
   - Key Features:
     - Real-time Ticker Search.
     - Beat Probability Gauges.
     - Key Signal Cards (Revenue Growth, Beat Rate, Last Surprise, Sentiment).
     - Interactive Price & Earnings History Charts (Recharts).
   - Deployment: Vercel.

B. BACKEND (API LAYER)
   - Framework: FastAPI (Python).
   - Responsibility: Orchestrates data fetching, feature engineering, 
     scoring calculation, and serving results via REST API.
   - Core Endpoints:
     - /predict/{ticker}: Main prediction engine.
     - /history/{ticker}: Historical EPS performance.
     - /signals/{ticker}: Detailed breakdown of momentum and sentiment.
     - /countdown/{ticker}: Next earnings date tracking.

C. DATA PROCESSING (FEATURE ENGINE)
   - Data Source: yfinance (Yahoo Finance API) for live prices, financials, 
     and news.
   - Modules:
     - per_quarter_features.py: Calculates growth, volatility, and momentum.
     - sentiment.py: Performs NLP-based sentiment analysis on company news.

D. INTELLIGENCE LAYER (SCORING & ML)
   - Research Model: Random Forest Classifier (trained on 10 years of FMP data).
   - Production Logic: Weighted Analytical Scoring System.
     - This system applies weights derived from the ML research phase:
       * Historical Beat Rate: 40%
       * Price Momentum (30d): 25%
       * Previous Surprise Momentum: 20%
       * Market Uncertainty (Volatility): 15%

3. DATA FLOW
------------
1. User enters a stock ticker in the React Frontend.
2. Frontend sends a GET request to the FastAPI Backend.
3. Backend triggers yfinance to fetch the latest financials, earnings 
   history, and recent news.
4. The Feature Engine processes raw data into normalized signals.
5. The Scoring System calculates the "Beat Probability" based on the signals.
6. Backend returns a JSON response containing the prediction, signals, 
   and chart data.
7. Frontend renders the results in a premium, interactive dashboard.

4. TECHNOLOGY STACK SUMMARY
---------------------------
- Frontend: React, Vite, Tailwind CSS, Lucide React, Recharts.
- Backend: Python, FastAPI, Uvicorn.
- Data Science: Pandas, NumPy, Scikit-Learn, yfinance, Joblib.
- Deployment: Vercel (Frontend), Render/DigitalOcean (Backend).
- Infrastructure: SQL (Original research), Local Pickled Models (Production).
